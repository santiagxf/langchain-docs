Middleware for models hosted on [Amazon Bedrock](https://aws.amazon.com/bedrock/). Learn more about [middleware](/oss/langchain/middleware/overview).

| Middleware | Description |
|------------|-------------|
| [Prompt caching](#prompt-caching) | Reduce costs by caching repetitive prompt prefixes |

## Prompt caching

Reduce inference latency and input token costs by caching frequently reused prompt prefixes on Amazon Bedrock. This pattern is useful for multi-turn agents with long system prompts, many tool definitions, or repeated document context.

Supported model families include **Anthropic Claude** and **Amazon Nova** on Bedrock.

<Info>
    Learn more about [AWS Bedrock prompt caching](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html) strategies, supported models, and regional limits.
</Info>

### LangChain agents

Agent middleware that places Bedrock cache checkpoints for `createAgent` is implemented in the [`langchain-aws`](https://github.com/langchain-ai/langchain-aws/tree/main/libs/aws) Python package (`langchain_aws.middleware.prompt_caching`). See that repository for installation, configuration options, and runnable examples.

For Bedrock chat models in TypeScript, start with the [Bedrock chat model integration](/oss/integrations/chat/bedrock) and the [@langchain/aws](https://www.npmjs.com/package/@langchain/aws) package.
